# Отчёт по безопасности Currency Exchange API

## Дата проверки : 15.05.2026

## 1. Аутентификация и авторизация

| Механизм | Реализация | Уровень |
|----------|------------|---------|
| Basic Auth | Для административных эндпоинтов | 🟡 Учебный |
| API Key | Для мобильных приложений | 🟡 Учебный |
| Ролевая модель | Администратор / Клиент | 🟢 Есть |

**Рекомендации для production:**
- Использовать JWT-токены вместо Basic Auth
- Интегрировать внешнюю систему пользователей
- Добавить двухфакторную аутентификацию

## 2. Защита от типовых уязвимостей

| Уязвимость | Защита | Статус |
|------------|--------|--------|
| SQL-инъекции | SQLAlchemy ORM (параметризованные запросы) | 🟢 Защищено |
| XSS | FastAPI автоматически экранирует JSON | 🟢 Защищено |
| CSRF | CORS ограничивает разрешённые домены | 🟢 Защищено |
| Переполнение буфера | Pydantic валидация входных данных | 🟢 Защищено |
| Race conditions | Unique constraints в БД | 🟢 Защищено |

## 3. Валидация входных данных

```python
# Пример валидации кода валюты
@field_validator('code')
def validate_code(cls, v: str) -> str:
    v = v.upper().strip()
    if not v.isalpha() or len(v) != 3:
        raise ValueError('Код валюты должен быть 3 заглавные буквы')
    return v
```

**Что проверяется:**
- Коды валют: ровно 3 буквы
- Суммы: положительные числа
- Курсы: положительные, до 6 знаков

## 4. Безопасность данных

| Данные | Хранение | Шифрование |
|---|---|---|
| Пароли админа | В переменных окружения (.env) | ❌ Нет (учебный проект) |
| Курсы валют | В SQLite (открытые) | ❌ Не требуется |
| Логи | На сервере | ❌ Не требуется |

**Для production:**
- Пароли хранить в хэшированном виде (bcrypt)
- Использовать HTTPS везде
- Настроить автоматическую ротацию логов

## 5. Мониторинг и логирование

**Что логируется:**
- Все запросы через uvicorn
- Ошибки приложения через исключения
- Действия администратора (можно добавить)

**Пример лога:**
`INFO:     127.0.0.1:54321 - "GET /exchange?from=USD&to=EUR&amount=100 HTTP/1.1" 200 OK`

## 6. Резервное копирование

**Скрипт:** scripts/backup.sh
**Расписание:** Ежедневно в 02:00 (cron)
**Хранение:** 30 последних бэкапов

## 7. Рекомендации по усилению безопасности

1. **Перевести на HTTPS** (Let's Encrypt + Nginx)
2. **Ограничить количество запросов** (rate limiting)
3. **Добавить веб-файервол** (ModSecurity)
4. **Регулярно обновлять зависимости** (`pip list --outdated`)
5. **Настроить мониторинг** (Prometheus + Grafana)

## Заключение

Проект демонстрирует базовые механизмы безопасности, достаточные для учебной практики.

Для production-использования требуется:
- HTTPS
- Хэширование паролей
- Rate limiting
- Аудит действий администратора
