## **Use-Case диаграмма: система «Обмен валют»** 

## **Акторы (кто взаимодействует)** 

## 1. **Клиент API** (Client) 

   - a. Любая программа или человек, отправляющий HTTP-запросы 

   - b. Не требует аутентификации 

   - c. Может читать курсы и выполнять конвертацию 

2. **Администратор** (Administrator) 

   - a. Управляет справочной информацией 

   - b. Добавляет новые валюты 

   - c. Добавляет и обновляет курсы обмена 

## **Варианты использования (что могут делать)** 

## **Клиент API** 

|**ID**|**Название**|**Краткое описание**|
|---|---|---|
|UC-01|Получить список всех валют|GET /currencies — возвращает массив всех валют|
|UC-02|Получить одну валюту по коду|GET /currency/USD — возвращает данные конкретной валюты|
|UC-03|Получить список всех курсов|GET /exchangeRates — все курсы с именами валют|
|UC-04|Получить курс по паре|GET /exchangeRate/USDEUR — курс между двумя валютами|
|UC-05|Конвертировать сумму|GET /exchange?from=USD&to=EUR&amount=100 — расчёт|

## **Администратор** 

|**ID**|**Название**|**Краткое описание**|
|---|---|---|
|UC-06|Добавить новую валюту|POST /currencies — создание валюты (с проверкой уникальности)|
|UC-07|Добавить курс обмена|POST /exchangeRates — создание курса (с проверкой уникальности пары)|
|UC-08|Обновить курс|PATCH /exchangeRate/USDEUR — изменение rate|

## **Связи между акторами** 

- Администратор **расширяет** возможности Клиента API (умеет всё то же самое + управление) 

- Клиент API **никак не влияет** на данные (только чтение и расчёт) 

## **Бизнес-правила (важно для понимания)** 

1. Код валюты — 3 заглавные буквы (USD, EUR, RUB) 

2. Курс обмена — положительное число с 6 знаками после запятой 

3. Одна пара валют может иметь только один курс (уникальность) 

4. При конвертации используются три сценария расчёта курса: 

   - a. Прямой курс (A→B есть в таблице) 

   - b. Обратный курс (B→A есть → 1/rate) 

   - c. Кросс-курс через USD (если есть A→USD и USD→B)
